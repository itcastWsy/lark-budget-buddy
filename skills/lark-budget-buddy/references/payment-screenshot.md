# 支付截图识别记账

> 本文档描述如何从用户在飞书聊天中发送的支付截图（微信/支付宝/银行App）自动提取交易信息并记账。

## 前置条件

确保已授权：
```bash
lark-cli auth login --domain im,base
```

所需 scope：
- `im:message:readonly` — 读取消息中的图片
- `im:message` — 发送确认消息
- `bitable:app` — 写入多维表格

## 工作流程

```
用户发送截图
    │
    ▼
Step 1: 获取消息 → 识别图片 → 下载图片
    │
    ▼
Step 2: AI 视觉分析图片 → 提取交易信息
    │
    ▼
Step 3: 展示解析结果 → 用户确认
    │
    ▼
Step 4: 写入多维表格
```

---

## Step 1: 获取并下载图片

当用户在飞书聊天中发送图片后，通过以下方式获取：

### 方式 A: 用户主动触发

用户在发送截图后说"记账"或"帮我记一下"，AI 执行：

```bash
# Step 1.1: 获取最近消息中的图片
lark-cli im +chat-messages-list \
  --chat-id <当前聊天ID> \
  --limit 5
```

在返回结果中找到 `msg_type: "image"` 的消息，提取：
- `message_id`（`om_xxx` 格式）
- `image_key`（`img_v3_xxx` 格式）

```bash
# Step 1.2: 下载图片
lark-cli im +messages-resources-download \
  --message-id <message_id> \
  --file-key <image_key> \
  --type image \
  --output ./payment_screenshot.png
```

### 方式 B: 事件监听自动触发

如果配置了 `lark-event` 事件订阅，可以自动监听新消息中的图片并触发处理。

---

## Step 2: AI 视觉分析截图

将下载的截图交给 AI 视觉模型分析，提取以下结构化信息：

### 提取目标

| 字段 | 说明 | 示例 |
|------|------|------|
| 金额 | 交易金额（数字） | 35.00 |
| 支付方向 | 支出/支出（微信/支付宝默认支出，退款/收入特殊标识） | 支出 |
| 商户/收款方 | 收款商户名称 | 沙县小吃 |
| 交易时间 | 交易发生时间 | 2026-04-15 12:30:00 |
| 支付方式 | 从截图来源判断 | 微信/支付宝/银行卡 |
| 交易类型 | 消费/转账/退款/充值等 | 消费 |
| 商品说明 | 具体的商品或服务描述 | 午餐 |

### 不同截图类型的识别要点

#### 微信支付截图特征
- 顶部绿色 "微信支付" 标识
- 金额通常以 ¥ 符号显示
- 底部显示"支付方式：零钱/银行卡"
- 关键字："当前状态"、"收款方"、"交易时间"

#### 支付宝截图特征
- 顶部蓝色 "支付宝" 标识
- 金额以 ¥ 符号显示
- 关键字："付款成功"、"向谁付款"、"付款时间"

#### 银行App截图特征
- 各银行UI不同，但核心信息一致
- 关键字："交易金额"、"收款人"、"交易时间"、"转账成功"

### AI 分类映射规则

从截图中提取的商户/商品描述，映射到记账分类：

| 商户关键词 | 映射分类 | 示例 |
|-----------|---------|------|
| 餐饮、食堂、奶茶、咖啡、美团外卖、饿了么、餐厅 | 餐饮 | 沙县小吃 → 餐饮 |
| 地铁、公交、滴滴、打车、高德打车、共享单车 | 交通 | 滴滴出行 → 交通 |
| 淘宝、京东、拼多多、超市、商场、便利店 | 购物 | 淘宝 → 购物 |
| 电影院、KTV、游乐园、景点门票、游戏充值 | 娱乐 | 万达影城 → 娱乐 |
| 房租、水电煤、燃气、宽带、物业费 | 居住 | 国家电网 → 居住 |
| 医院、药房、药店、体检、诊所 | 医疗 | 和睦家医院 → 医疗 |
| 学校、培训、教育、网课、教材、书店 | 教育 | 新东方 → 教育 |
| 中国移动、联通、电信、话费 | 通讯 | 中国移动 → 通讯 |
| 日用品、超市日用品、生活用品 | 日用 | 沃尔玛日用品 → 日用 |
| 工资、奖金、绩效 | 工资（收入） | 公司代发工资 → 收入/工资 |
| 理财收益、基金、股票收益 | 理财（收入） | 基金分红 → 收入/理财 |

## Step 3: 展示解析结果并确认

分析完成后，向用户展示解析结果，等待确认：

```
📸 已识别支付截图：

| 项目 | 内容 |
|------|------|
| 金额 | ¥35.00 |
| 类型 | 支出 |
| 分类 | 餐饮（自动识别：沙县小吃） |
| 商户 | 沙县小吃 |
| 时间 | 2026-04-15 12:30 |
| 支付方式 | 微信 |
| 交易类型 | 消费 |

确认记录？（回复"确认"或"修改XX为XX"）
```

**如果用户确认：** 执行 Step 4
**如果用户要求修改：** 修改对应字段后确认写入

---

## Step 4: 写入多维表格

```bash
lark-cli base +record-upsert \
  --base-token WrzBbBAvNaU83BsVNt0ci7c9ndf \
  --table-id tblkghyZJ3LiZ6Z1 \
  --json '{
    "日期": "2026-04-15 12:30:00",
    "类型": "支出",
    "金额": 35,
    "分类": "餐饮",
    "备注": "沙县小吃 午餐",
    "支付方式": "微信"
  }'
```

写入成功后回复：
```
✅ 已记录：支出 ¥35.00（餐饮 - 沙县小吃 午餐 - 微信）
```

## 异常处理

### 截图模糊/无法识别
```
⚠️ 截图不够清晰，无法识别交易信息。
请发送更清晰的截图，或手动输入（如：午饭 35 微信）
```

### 无法判断分类
```
📸 已识别：支出 ¥128.00（商户：XXXXXX）
⚠️ 无法自动判断消费分类，请指定分类：
   回复示例："分类 餐饮" / "分类 购物" / "分类 其他"
```

### 多条支付记录在同一截图
```
📸 截图包含多笔交易：
1. ¥35.00 - 沙县小吃
2. ¥12.00 - 蜜雪冰城

是否全部记录？（回复"是"或选择记录哪些）
```

### 截图不是支付类
```
⚠️ 未检测到支付相关信息。
请发送微信/支付宝/银行App的支付截图，或手动输入记账信息。
```
